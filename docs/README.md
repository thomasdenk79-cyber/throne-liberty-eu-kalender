# Dokumentations- und Wissensstruktur

Diese Dokumentation ist das gemeinsame Projektgedaechtnis fuer Mensch, ChatGPT und Coding-Agenten.

## Einstieg

| Bereich | Zweck |
|---|---|
| `docs/project/` | Was ist das Projekt, warum gibt es das, wie ist es aufgebaut? |
| `docs/engineering/` | Wie wird entwickelt, getestet, dokumentiert und mit KI gearbeitet? |
| `docs/demo/` | Material fuer die Vorstellung bei Kollegen und als Kopiervorlage fuer andere Projekte |

## Schnellstart fuer Agenten

1. `AGENTS.md` lesen.
2. Diese Datei lesen.
3. Je nach Aufgabe gezielt in `docs/project/` oder `docs/engineering/` einsteigen.
4. Umsetzung klein halten und Dokumentation aktualisieren.
5. Entscheidungen im `docs/engineering/decision-log.md` nachtragen.

## Ziel der Struktur

Die Dateien sollen nicht nur Menschen helfen, sondern auch KI-Agenten. Ein Agent soll nach dem Klonen des Repositories schnell verstehen:

- wofuer die Seite gedacht ist,
- welche technischen Grenzen bewusst gesetzt sind,
- wie Aenderungen sicher umgesetzt werden,
- welche Entscheidungen bereits getroffen wurden,
- wie dieses kleine Projekt als Demo fuer KI-gestuetzte Entwicklung genutzt werden kann.

## Abgrenzung

Dieses oeffentliche Privat-Repository enthaelt keine vertraulichen Unternehmensdaten. Beispiele aus beruflichen Kontexten duerfen nur abstrahiert und ohne interne Namen, Systeme, Daten oder Zugangsinformationen dokumentiert werden.
